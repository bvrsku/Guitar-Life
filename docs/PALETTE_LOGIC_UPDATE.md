# Новая логика палитр в Guitar Life

## Обзор

Я успешно реализовал улучшенную логику работы с палитрами AGE и RMS в Guitar Life. Теперь система поддерживает гибкое управление палитрами с различными режимами смешивания.

## Основные возможности

### 1. Режимы работы RMS палитры

**`Brightness` (Яркость)**
- RMS влияет на яркость и насыщенность клетки
- Основной цвет берется от AGE палитры
- Совместимо с предыдущей версией

**`Palette` (Палитра)**
- RMS палитра смешивается с AGE палитрой
- Поддерживает различные режимы блендинга
- Сила смешивания зависит от RMS значения

**`Disabled` (Отключено)**
- RMS палитра полностью отключена
- Используется только AGE палитра
- RMS не влияет на цвет

### 2. Режимы блендинга палитр

При использовании режима `Palette`, доступны следующие режимы смешивания:

**`Normal`** - Альфа-блендинг (линейная интерполяция)
```
Результат = AGE_цвет * (1 - фактор) + RMS_цвет * фактор
```

**`Additive`** - Аддитивное смешивание
```
Результат = min(255, AGE_цвет + RMS_цвет * фактор)
```

**`Screen`** - Режим экрана (осветление)
```
Результат = 255 - (255 - AGE_цвет) * (255 - RMS_цвет * фактор) / 255
```

**`Multiply`** - Умножение (затемнение)
```
Результат = AGE_цвет * (RMS_цвет * фактор / 255 * фактор + (1 - фактор))
```

**`Overlay`** - Перекрытие (контраст)
```
Если база < 128: 2 * база * наложение / 255
Иначе: 255 - 2 * (255 - база) * (255 - наложение) / 255
```

### 3. Элементы управления в GUI

Для каждого слоя добавлены новые элементы:

- **RMS Mode**: Выбор режима (`Brightness`, `Palette`, `Disabled`)
- **Blend Mode**: Режим блендинга (`Normal`, `Additive`, `Screen`, `Multiply`, `Overlay`)
- **RMS Enabled**: Кнопка включения/выключения RMS палитры

## Логика работы

### Схема принятия решений:
```
RMS включена? (rms_enabled)
├─ НЕТ → Только AGE палитра
└─ ДА → Какой режим RMS?
   ├─ "disabled" → Только AGE палитра
   ├─ "brightness" → AGE палитра + RMS модулирует яркость
   └─ "palette" → Смешивание палитр с выбранным блендингом
```

### Пример использования:

1. **Классический режим** (как раньше):
   - RMS Mode: `Brightness`
   - RMS Enabled: `Включено`

2. **Только возраст**:
   - RMS Mode: `Disabled` или RMS Enabled: `Выключено`

3. **Смешивание палитр**:
   - RMS Mode: `Palette`
   - Blend Mode: выбрать подходящий (`Normal`, `Additive`, и т.д.)
   - RMS Enabled: `Включено`

## Технические детали

### Новые функции:

- `blend_colors()` - универсальная функция смешивания цветов
- `color_from_age_only()` - цвет только от возраста
- `color_from_age_brightness_rms()` - старая логика для режима brightness

### Обновленные структуры:

```python
@dataclass
class Layer:
    # ... существующие поля ...
    rms_mode: str = "brightness"  # "brightness" | "palette" | "disabled"
    blend_mode: str = "normal"    # "normal" | "additive" | "screen" | "multiply" | "overlay"
    rms_enabled: bool = True      # Включена ли RMS палитра
```

## Результат

Теперь у вас есть полный контроль над тем, как палитры AGE и RMS взаимодействуют:

1. **Отключить RMS**: Установите `RMS Enabled = false` или `RMS Mode = Disabled`
2. **Классическое поведение**: `RMS Mode = Brightness`
3. **Творческое смешивание**: `RMS Mode = Palette` + выбор `Blend Mode`

Все изменения обратно совместимы с существующими настройками.
