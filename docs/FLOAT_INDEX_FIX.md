# Исправление ошибки "list indices must be integers or slices, not float"

## ✅ Проблема решена

**Ошибка**: `Fatal: list indices must be integers or slices, not float`

**Причина**: В функции `build_color_image()` использовались индексы из `np.nonzero()` без явного преобразования в `int`. В некоторых случаях NumPy может возвращать индексы с типом данных, который интерпретируется как float.

## 🔧 Исправление

**Файл**: `guitar_lifeeq.py`  
**Функция**: `build_color_image()` (строки ~2997-3012)

### До исправления:
```python
# быстрый проход по True-ячейкам
ys, xs = np.nonzero(layer_grid)
for idx, (i, j) in enumerate(zip(ys, xs)):
    try:
        # ... код обработки цвета ...
        img[i, j] = color  # ❌ Ошибка: i, j могут быть float
```

### После исправления:
```python
# быстрый проход по True-ячейкам
ys, xs = np.nonzero(layer_grid)
for idx, (i, j) in enumerate(zip(ys, xs)):
    # Преобразуем индексы в int для безопасности
    i, j = int(i), int(j)  # ✅ Исправлено
    try:
        # ... код обработки цвета ...
        img[i, j] = color  # ✅ Теперь безопасно
```

## 📊 Техническая информация

**Типы данных**: `np.nonzero()` иногда возвращает индексы с типом, который может быть интерпретирован как float в зависимости от контекста массива.

**Решение**: Явное преобразование `int(i), int(j)` гарантирует что индексы всегда будут целыми числами.

**Производительность**: Минимальное влияние на производительность, добавляется только одна операция преобразования на итерацию.
