# HUD Performance Optimizations Summary

## Проблема
Пользователь сообщил: "HUD очень жрет производительность. Когда я нажимаю хоткей H, HUD становится недоступным для редактирования, а количество кадров повышается в сотни раз"

## Примененные оптимизации

### 1. Кэширование поверхности HUD
- **Переменные кэша:**
  - `hud_surface_cache` - кэшированная поверхность HUD
  - `hud_cache_valid` - флаг валидности кэша  
  - `hud_last_update` - время последнего обновления

- **Принцип работы:** HUD отрисовывается в отдельную поверхность только при изменениях, затем эта поверхность просто копируется на экран

### 2. Ограничение частоты обновления HUD
- **HUD обновляется максимум 30 FPS** вместо 60 FPS основного рендеринга
- **Информация info обновляется с разной частотой:**
  - Быстрые данные (RMS, Pitch, Tick): каждый кадр (60 FPS)
  - Медленные данные (подсчет клеток, альфа-значения, FX): 10 раз в секунду

### 3. Умная инвалидация кэша
Кэш автоматически инвалидируется при:
- Любом изменении параметров через `on_hud_parameter_change()`
- Переключении видимости HUD (клавиша H)
- Изменении FX эффектов (F1, F2, F3)
- Изменении правил слоев (`cycle_layer_rule()`)

### 4. Условная обработка событий
- **События HUD обрабатываются только когда HUD видим**
- Код: `if self.hud.visible and self.hud.handle_event(ev):`

### 5. Оптимизация создания info
```python
# Быстрые данные - каждый кадр
fast_info = {
    "RMS": f"{rms:.4f}",
    "Pitch": f"{pitch:.1f} Hz" if pitch > 0 else "—",
    "Tick": f"{dyn_ms} ms",
}

# Медленные данные - кэшируются на 100ms
if current_time - self._info_last_update >= self._info_update_interval:
    self._cached_info.update({
        "Alive": f"{total_alive} cells",
        # ... другие тяжелые вычисления
    })
```

## Ключевые методы

### `invalidate_hud_cache()`
Принудительно инвалидирует кэш HUD для случаев когда нужно гарантированно обновить отображение.

### Оптимизированный рендеринг HUD
```python
if self.hud.visible:
    needs_update = (
        current_time - self._hud_last_update >= self._hud_update_interval or
        not self.hud_cache_valid or
        self.hud_surface_cache is None
    )
    
    if needs_update:
        # Отрисовываем в кэш
        self.hud_surface_cache.fill((0, 0, 0, 0))
        self.hud.draw(self.hud_surface_cache, info)
        self.hud_cache_valid = True
    
    # Всегда используем кэш для отображения
    self.screen.blit(self.hud_surface_cache, (self.W, 0))
```

## Результат
Теперь HUD обновляется только при реальных изменениях параметров, а не каждый кадр, что должно значительно снизить нагрузку на производительность. Система кэширования гарантирует что:

1. **HUD не тормозит основной рендеринг** - работает на отдельной частоте 30 FPS
2. **События обрабатываются эффективно** - только когда HUD видим
3. **Информация обновляется оптимально** - тяжелые расчеты кэшируются
4. **Кэш инвалидируется своевременно** - при всех изменениях которые влияют на отображение

## Замечания
- Все оптимизации обратно совместимы
- Функциональность HUD не изменилась
- Добавлен метод `invalidate_hud_cache()` для ручной инвалидации при необходимости
