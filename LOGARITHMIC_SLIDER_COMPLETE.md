## Техническая реализация

### Алгоритм логарифмического масштабирования

**Первая половина (0-50%):**
```python
if slider_percent <= 50.0:
    t = slider_percent / 50.0  # 0.0 - 1.0
    return int(10 + t * 60)    # 10-70
```

**Вторая половина (50-100%):**
```python
else:
    t = (slider_percent - 50.0) / 50.0  # 0.0 - 1.0
    log_t = math.pow(t, 1.5)            # Логарифмическое ускорение
    return int(70 + log_t * 430)        # 70-500
```

### Распределение значений

| Позиция слайдера | Max Age | Описание |
|------------------|---------|----------|
| 0% | 10 | Минимум - очень быстрая смерть |
| 25% | 40 | Четверть первой половины |
| 50% | 70 | Граница между половинами |
| 75% | ~222 | Логарифмическое ускорение |
| 100% | 500 | Максимум - очень долгая жизнь |

### Преимущества логарифмического масштабирования

1. **Точный контроль малых значений**: Первая половина слайдера дает подробный контроль в диапазоне 10-70
2. **Доступ к большим значениям**: Вторая половина покрывает широкий диапазон 70-500
3. **Интуитивное использование**: Большинство полезных значений находятся в первой половине
4. **Плавное масштабирование**: Нет резких скачков между значениями
